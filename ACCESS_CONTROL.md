# نظام التحكم بالوصول - نظام جرد حلويات الشيخ

## نظرة عامة

النظام يوفر تحكماً كاملاً بالوصول بحيث يكون خاصاً بك فقط. فقط المستخدمون المسموح لهم يمكنهم الدخول والوصول إلى البيانات.

## الأدوار والصلاحيات

| الدور | الصلاحيات |
|------|----------|
| **مالك (Owner)** | - الوصول الكامل للنظام<br>- إضافة/حذف المستخدمين<br>- عرض سجلات الوصول<br>- إدارة الإعدادات |
| **مدير (Manager)** | - الوصول الكامل للعمليات<br>- عرض التقارير<br>- إضافة عمليات جديدة |
| **موظف (Employee)** | - إضافة عمليات جديدة فقط<br>- عرض الجرد اليومي |

## خطوات الإعداد الأولي

### 1. إضافة المستخدم الأول (المالك)

عند النشر لأول مرة:

1. اذهب إلى صفحة إدارة المستخدمين: `/admin/users`
2. أضف حسابك كمالك:
   - البريد الإلكتروني: بريدك الإلكتروني
   - الاسم: اسمك
   - الدور: مالك (Owner)

### 2. إضافة مستخدمين إضافيين

من صفحة الإدارة:

1. انقر على **إضافة مستخدم جديد**
2. أدخل:
   - البريد الإلكتروني للمستخدم
   - اسم المستخدم
   - اختر الدور المناسب
3. انقر **إضافة**

## سجل محاولات الوصول

يتم تسجيل كل محاولة وصول (سواء مسموح أو مرفوض) مع:
- البريد الإلكتروني
- حالة الوصول (مسموح/مرفوض)
- السبب (إن وجد)
- عنوان IP
- التاريخ والوقت

## الميزات الأمنية

✅ **قائمة بيضاء (Whitelist):** فقط المستخدمون المضافون يمكنهم الدخول

✅ **تسجيل محاولات الوصول:** كل محاولة تُسجل للمراجعة

✅ **الأدوار والصلاحيات:** كل مستخدم له صلاحيات محددة

✅ **تفعيل/تعطيل المستخدمين:** يمكن تعطيل حساب دون حذفه

## أمثلة الاستخدام

### مثال 1: إضافة موظف جديد

```
البريد الإلكتروني: ahmed@example.com
الاسم: أحمد محمد
الدور: موظف
```

أحمد سيتمكن من:
- إضافة عمليات جديدة
- عرض الجرد اليومي

### مثال 2: إضافة مدير

```
البريد الإلكتروني: manager@example.com
الاسم: محمد علي
الدور: مدير
```

المدير سيتمكن من:
- إضافة عمليات جديدة
- عرض جميع التقارير
- إدارة البيانات

## استكشاف الأخطاء

### المشكلة: "وصول مرفوض"

**الحل:**
1. تأكد من إضافة بريدك الإلكتروني في قائمة المستخدمين المسموح لهم
2. تأكد من أن الحساب نشط (Active)
3. اتصل بالمالك لطلب الوصول

### المشكلة: لا يمكن إضافة مستخدمين جدد

**الحل:**
1. تأكد من أن حسابك هو "مالك" (Owner)
2. فقط المالك يمكنه إضافة مستخدمين جدد

## الخصوصية والأمان

- جميع البيانات مشفرة
- سجلات الوصول محفوظة بأمان
- لا يمكن حذف سجلات الوصول (للمراجعة)
- كل عملية تُسجل مع معلومات المستخدم

## الدعم

للمساعدة في إدارة الوصول، يرجى التواصل مع المالك.
